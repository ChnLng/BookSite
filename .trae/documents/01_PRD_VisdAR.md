# Visd AR 电子书独立站 PRD（产品需求文档）

## 1. 项目概述

### 1.1 目标
- 搭建品牌 Visd AR 的独立站，销售当前在 Amazon Author Store（4 本电子书）上的同款电子书，并支持后续新增图书与类目扩展。
- 站点面向用户端全法语（FR）。
- 管理员后台为汉语 + 法语双语展示（同屏双语，不使用“/”分隔）。
- 收款使用 Stripe（支持电子书购买与自由 Donation “买我一杯咖啡”）。
- 数据库与鉴权使用 Supabase。

### 1.2 受众
- 终端读者（法语用户为主）
- 管理员（运营/内容管理，偏中文使用习惯但需法语对照）

### 1.3 成功标准
- 用户可用 Google / GitHub / 邮箱注册登录，购买电子书并查看购买记录。
- 管理员可上架新书、管理类目、库存、价格、图片、优惠季与优惠码。
- 支付成功后可安全交付电子书（可邮件通知到 visdar@outlook.fr），并具备退款/拒付后的访问防御与追踪机制。
- 页面“结构越简单越好”，内容不溢出屏幕，尽量减少滚动（桌面优先 + 移动端兼容）。

## 2. 品牌与视觉

### 2.1 品牌
- 品牌名：Visd AR
- Logo：使用用户提供的两张图，优先使用图二（猫咪眼睛图形）

### 2.2 视觉风格方向
- 清透感、轻盈、精致、有互动
- 避免内容溢出与长滚动：采用“单屏主舞台 + 分区切换（Tabs/Drawer/Modal）”
- 动效：微交互为主（悬停、购买反馈、优惠倒计时、推荐卡片翻转等）

## 3. 信息架构（用户端 FR）

### 3.1 顶部导航（固定）
- Accueil
- Livres
- Recommandations
- Donation
- Mon compte
- Support

### 3.2 首页（单屏）
- Hero：品牌 + 简介 + 4 本书快捷卡片（可切换查看详情）
- 促销条：当前优惠季、倒计时、优惠码一键复制
- 推荐区域：基于“读者偏好/热销/同系列”的精选推荐卡片
- CTA：查看所有图书 / 登录 / Donation

### 3.3 图书列表（尽量单屏）
- 类目筛选（Catégories）
- 排序（Nouveautés / Popularité / Prix）
- 图书卡片：封面、标题、价格、库存状态、折扣信息

### 3.4 图书详情（Modal 或独立页）
- 基本信息：标题、简介、格式（PDF/EPUB/MOBI）、语言、页数（可选）
- 价格与折扣：原价/折扣价/优惠提示
- 购买按钮：Acheter
- 纸质书引导按钮：Version papier sur Amazon（跳转 Amazon 对应链接）
- 推荐：Vous aimerez aussi

### 3.5 购物与交付
本项目优先采用“单本购买”简化结构（也可扩展为购物车）。
- Stripe Checkout：用户选择购买后跳转支付
- 支付完成页：显示下载入口（受保护）+ 邮件确认提示
- 我的账户：查看购买记录、可再次下载（受限次数/有效期）

### 3.6 Donation（买我一杯咖啡）
- 预设金额（例如 3€ / 5€ / 10€）+ 自定义金额
- Stripe Checkout 支付
- 感谢页：可选留言（与消息系统共用）

### 3.7 留言与支持（Support）
两类入口（合并 UI）：
- Contact：用户发送留言（问题/合作/反馈）
- Commentaire：对站点或书籍的公开/半公开留言（按产品关联，可选）

## 4. 用户系统与权限

### 4.1 登录方式
- Google OAuth
- GitHub OAuth
- 邮箱注册登录（magic link 或密码登录，具体按实现选择）

### 4.2 用户权限
- Visitor：未登录，只能浏览
- User：登录后可购买、查看购买记录、下载已购内容、留言
- Admin：后台管理（可配置多个管理员）

## 5. 管理员后台（ZH + FR 同屏）

### 5.1 核心模块
- 仪表盘：订单/收入/本周销量/退款与拒付提醒
- 图书管理（Livres 管理）
  - 新增图书（标题、简介、类目、格式、库存、价格、封面与文件）
  - 编辑图书：可手动修改 图片名称、数量、价格、库存
  - 上下架
- 类目管理（Catégories）
  - 新建/编辑/排序
- 库存管理（Stock）
  - 电子书可用“库存=无限/限量”两种模式；需要一个统一字段支持
- 优惠与营销（Promotions）
  - 优惠季（开始日期、结束日期、展示文案、倒计时开关）
  - 优惠码（code、折扣类型、折扣值、适用范围、有效期、使用次数）
- 订单与交付（Commandes）
  - 订单列表、状态、用户、商品、金额、Stripe 状态
  - 交付状态：下载次数、最近下载时间、是否被锁定
- 留言管理（Messages）
  - 查看/标记已处理/导出
- 推荐管理（Recommandations）
  - 手动设定推荐（同类/同系列/热销）

### 5.2 双语展示规则（不使用斜杠）
- 同屏双语：例如“库存 Stock”“价格 Prix”
- 重要字段可用“中文（Français）”或“中文  Français”形式

## 6. 退款/拒付防御（核心策略）

### 6.1 风险场景
- 用户购买后立即下载并发起退款/信用卡拒付（chargeback）
- 多账号薅下载、分享链接、盗版传播

### 6.2 防御目标
- 退款/拒付后即时撤销下载权限
- 可追溯下载行为（IP、时间、次数）
- 降低盗链与外泄风险（链接过期、签名下载、限次）

### 6.3 方案要求（MVP）
- 下载必须登录 + 已购校验
- 下载链接为短时效签名 URL（例如 5–30 分钟）
- 每个订单限制下载次数（例如 3–5 次）与/或限制下载有效期（例如 7 天）
- Stripe Webhook 监听退款、争议（dispute）事件：
  - 自动将订单标记为 refunded / disputed
  - 立即锁定该订单的下载权限
- 水印（可选增强）：
  - 在 PDF 内嵌“购买邮箱 + 订单号 + 日期”水印
  - 或在下载包内附带“购买凭证文本文件”

## 7. 数据与内容来源
- 当前四本电子书：来自 Amazon Author Store 的同款商品信息（标题、封面、简介、纸质书链接）
- 站内销售电子书文件由管理员上传（Supabase Storage）

## 8. 非功能性需求
- 性能：首屏快速加载、图片自适应与压缩
- SEO：法语 SEO（title/description、OG、结构化数据）
- 可访问性：基础键盘可用性、按钮对比度
- 安全：RLS、最小权限、Webhook 校验、敏感密钥仅服务端使用

## 9. 范围与里程碑（按实现阶段）
- Phase 1（MVP）：图书展示 + 登录 + Stripe 支付 + 交付下载 + 后台管理 + 留言
- Phase 2：推荐系统增强、优惠季与多种促销玩法、下载水印
- Phase 3：小游戏入口（独立模块，可复用同账号体系）
